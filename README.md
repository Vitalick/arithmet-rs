# Устный счет на Rust

`arithmet` - терминальная программа для тренировки устного счета. Это Rust-
реализация идеи из C-проекта
[arithmet_c](https://github.com/Vitalick/arithmet_c.git): пользователь выбирает
арифметические действия и параметры тренировки, решает примеры на время, затем
получает оценку и сохраненный протокол.

## Возможности

- адаптивный TUI на `ratatui` и `crossterm`;
- выбор действий: сложение, вычитание, умножение, деление и деление с остатком;
- настройка имени игрока, границ результата, количества примеров и времени на
  ответ;
- хранение настроек в `arithmet.toml`;
- генерация упражнений и проверка ответов в доменном слое без привязки к TUI;
- оценка по пятибалльной шкале;
- сохранение каждого завершенного сеанса в `results/<user>/` сразу в двух
  форматах: JSON для программы и TXT для чтения человеком;
- просмотр результатов последнего завершенного сеанса внутри приложения.

## Управление

- `+`, `-`, `*`, `/`, `:` - включить или выключить действие;
- `И` - изменить имя;
- `О` - изменить нижнюю границу результата;
- `Д` - изменить верхнюю границу результата;
- `К` - изменить количество примеров;
- `С` - изменить время на ответ;
- `Enter` - начать сеанс, подтвердить ввод или перейти к следующему примеру;
- `Tab` - перейти к следующему примеру после проверки ответа;
- `F1` - открыть результаты последнего завершенного сеанса;
- `Esc` - отменить ввод, выйти из результатов или завершить текущий режим;
- `Q` или `Ctrl+C` - выйти из приложения.

## Настройки

Пример `arithmet.toml`:

```toml
player_name = "неизвестно"
results_dir = "results"
operations = ["+", "-", "*", "/", ":"]

[limits]
result_min = 100
result_max = 150
exercise_count = 5
answer_time = "10s"
```

`answer_time` задает текущий общий лимит времени на один ответ. В интерфейсе
этот параметр пока подписан как `Сложность`; в плане развития он будет
переименован во `Время на ответ`.

## Результаты

После завершения сеанса программа создает каталог для игрока внутри
`results_dir` и сохраняет файлы вида:

```text
results/<user>/2026-07-01_12-30-45.json
results/<user>/2026-07-01_12-30-45.txt
```

JSON хранит полные данные сеанса и будет использоваться для будущего просмотра
старых запусков из приложения. TXT остается удобным текстовым протоколом.

## Планы

Актуальные планы лежат в [docs/ROADMAP.md](docs/ROADMAP.md). Ближайшие темы:
управление мышью, просмотр результатов из файловой системы, переименование
сложности во время на ответ, разные лимиты времени для разных действий и новые
операции `^`, корень и `%`.

## Сборка и запуск

```bash
cargo build
cargo run
```
